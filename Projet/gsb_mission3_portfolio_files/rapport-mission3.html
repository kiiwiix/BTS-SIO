GSB · Mission 3

Mise en place d’une infrastructure Active Directory, DNS et DHCP

Cette mission complète les missions 1 et 2 : après la segmentation VLAN et le routage via Stormshield, les services centraux du domaine GSB sont mis en place pour authentifier les utilisateurs, résoudre les noms et distribuer automatiquement les paramètres IP.

← Retour au portfolio

Contexte

Objectif de la mission

Le réseau GSB dispose déjà de VLAN et d’une sortie Internet via le pare-feu. L’objectif de cette troisième mission est d’ajouter les services Windows Server nécessaires à un environnement professionnel : annuaire Active Directory, DNS interne et DHCP.

Résultat attendu : un domaine fonctionnel, une résolution DNS interne opérationnelle et une configuration IP distribuée automatiquement aux postes clients.

Problème

Besoin identifié

Sans Active Directory, la gestion des comptes et des postes reste locale. Sans DNS interne fiable, les machines ne retrouvent pas correctement les services du domaine. Sans DHCP, chaque client doit être configuré manuellement.

Solution

Réponse technique

Déploiement d’un socle Windows Server avec Active Directory, DNS intégré et DHCP. La validation est faite depuis un client Windows 10 connecté au réseau GSB.

Réalisation

Étapes principales

1

Préparation des serveurs

Configuration réseau des serveurs, nommage cohérent et vérification de la connectivité avec le VLAN concerné.

2

Installation d’Active Directory

Ajout du rôle AD DS puis promotion du serveur en contrôleur de domaine pour le domaine swiss-galaxyB2.com.

3

Configuration DNS

Mise en place du DNS intégré à Active Directory. Le client utilise les serveurs DNS internes 192.168.250.20 et 192.168.250.21.

4

Configuration DHCP

Le serveur DHCP distribue automatiquement l’adresse IP, la passerelle 192.168.250.254, le suffixe DNS swiss-galaxyB2.com et les DNS internes.

5

Tests client

Validation depuis un client Windows 10 avec ipconfig /all : bail DHCP obtenu, passerelle correcte, DNS internes visibles et suffixe DNS du domaine présent.

Conclusion

Bilan

La mission 3 ajoute le socle serveur indispensable au réseau GSB. Les postes peuvent utiliser le domaine, résoudre les noms internes et recevoir automatiquement leur configuration réseau.

À expliquer à l’oral : Active Directory centralise les identités, DNS permet au domaine de fonctionner correctement, et DHCP évite les erreurs de configuration manuelle sur les postes.
