Contexte

Présentation de l'entreprise

Adefi Informatique est une société de services informatiques spécialisée dans l'assistance aux entreprises et aux collectivités locales. Elle intervient sur l'ensemble du parc matériel de ses clients : postes de travail, serveurs, équipements réseau (switchs, routeurs, pare-feux) et solutions NAS.

L'entreprise assure également le support, la maintenance préventive et curative, ainsi que l'administration des infrastructures Windows Server / Active Directory de ses clients.

Semaine 1

Découverte & premières interventions

Activités principales

  • Remplacement de disques et mise en place d'un RAID 1 sur un serveur.
  • Élaboration de devis pour renouvellement de postes obsolètes.
  • Accueil clientèle & diagnostic rapide.
  • Migration d'un poste Windows 10 vers Ubuntu.
  • Configuration d'un NAS Synology avec RAID SHR.
  • Installation complète de Windows 11 (pilotes, nettoyage, restauration de données).
Difficulté rencontrée : Performances insuffisantes d'un disque dur mécanique → diagnostic via CrystalDiskInfo → remplacement par SSD et remontée du RAID.

Ce que j'ai appris

  • Bases du diagnostic matériel (disques, RAM, alimentation).
  • Découverte des environnements Linux (Ubuntu) et Synology DSM.
  • Importance du RAID et des sauvegardes pour les entreprises.

Captures à intégrer

NAS Synology — Interface DSM (RAID SHR)

Cette capture montre le NAS Synology entièrement configuré avec le RAID SHR actif et les disques en bonne santé.

📸
Interface DSM — Gestionnaire de stockage

Quoi capturer : Panneau de configuration DSM → Gestionnaire de stockage → Pool de stockage. On doit voir le RAID SHR avec le statut « En bonne santé » et les disques listés avec leur capacité.

  1. Connecte-toi à l'interface web du NAS (ex. 192.168.x.x:5000)
  2. Panneau de configuration → Gestionnaire de stockage
  3. Onglet "Pool de stockage" ou "Volume"
  4. Capture l'écran montrant le RAID SHR actif

Gestionnaire de disques Windows — RAID 1 serveur

Preuve de la mise en place du RAID 1 sur le serveur Windows, avec les deux disques en miroir.

📸
Gestionnaire de disques Windows Server

Quoi capturer : Ouvre le Gestionnaire de disques (diskmgmt.msc). On doit voir les deux disques du RAID 1 avec la mention « Miroir » ou « En miroir » et le statut « Sain ».

  1. Win+R → diskmgmt.msc
  2. Localise les volumes en miroir (RAID 1)
  3. Capture l'écran complet montrant les deux disques
Semaine 2

Infrastructure réseau & pare-feu

Activités principales

  • Étude & utilisation d'iptables (pare-feu Linux).
  • Configuration d'un pare-feu matériel, d'un switch Zyxel et d'un routeur.
  • Déploiement d'une infrastructure réseau complète dans un bâtiment professionnel.

  • Réparation d'un PC instable (diagnostic alimentation).
Compétences acquises : Mise en place de VLANs, adresses IP fixes, gestion des conflits réseau (ports téléphonie vs data), techniques de test matériel (alimentation).

Captures à intégrer

Règles iptables — Pare-feu Linux

Capture des règles iptables configurées, montrant les chaînes INPUT/OUTPUT/FORWARD et les règles de filtrage mises en place.

📸
Terminal Linux — iptables -L -v

Quoi capturer : Terminal Linux affichant la liste des règles iptables avec les compteurs de paquets et les politiques par chaîne.

  1. Ouvre un terminal Linux (ou une VM si plus en stage)
  2. Tape : sudo iptables -L -v --line-numbers
  3. Capture le résultat montrant les chaînes INPUT, FORWARD, OUTPUT

Switch Zyxel — Configuration VLAN

Interface d'administration du switch Zyxel montrant les VLANs configurés pour séparer les flux téléphonie et données.

📸
Interface web du switch Zyxel

Quoi capturer : Interface d'administration web du switch → Section VLAN. On doit voir les VLANs créés avec leurs ports assignés (accès ou trunk).

  1. Accède à l'interface web du switch Zyxel (IP de gestion)
  2. Navigue vers Advanced → VLAN → Static VLAN
  3. Capture l'écran avec les VLANs et leurs configurations de ports
📸
Photo de la baie informatique brassée

Quoi capturer : Photo physique de la baie réseau après brassage complet — câbles organisés, switchs en place, serveurs montés. Si tu n'as pas la photo, un schéma de baie dessiné à la main scanné suffit.

Semaine 3

Ateliers techniques & dépannage avancé

Activités principales

  • Récupération de données d'un PC bloqué via machine virtuelle Fedora.
  • Réinitialisation complète d'un téléphone Samsung.
  • Recherche et installation manuelle de pilotes inconnus.
  • Suppression d'un mot de passe Windows via outil bootable.
  • Prise en main à distance d'un serveur avec TeamViewer.
  • Montage d'un PC Gamer (2 400 €) avec watercooling.
  • Intervention en entreprise sur un NAS défaillant.
Technique clé : Utilisation d'une VM Fedora pour monter le disque du PC bloqué et accéder aux données sans démarrer le système d'exploitation corrompu.

Ce que j'ai appris

  • Analyse des droits Windows et contournement sécurisé via outil bootable.
  • Utilisation d'une VM Linux pour le dépannage hors-système.
  • Montage avancé de PC haute performance (watercooling, gestion des câbles).
  • Diagnostic alimentation, RAM, stockage.

Captures à intégrer

VM Fedora — Récupération de données

La machine virtuelle Fedora montée sur VirtualBox/VMware, avec le disque du PC client accessible et les fichiers visibles dans le gestionnaire de fichiers.

📸
VM Fedora avec disque client monté

Quoi capturer : Fenêtre VirtualBox / VMware avec Fedora démarré, gestionnaire de fichiers ouvert et les fichiers du PC client visibles. Si tu n'as plus accès, reproduis la démarche : monte un disque externe dans une VM Linux et capture le résultat.

  1. Lance VirtualBox / VMware → Ouvre ta VM Linux
  2. Attache un disque via "Périphériques → USB" ou SATA
  3. Ouvre le gestionnaire de fichiers Fedora/Nautilus

PC Gamer assemblé — Photo physique

Quoi capturer : Photo du PC Gamer monté avec le watercooling en place, câbles gérés proprement, vu de l'intérieur du boîtier. Si tu as la photo prise pendant le stage, intègre-la directement ici.

Captures à intégrer

Commandes réseau — arp -a & ipconfig

Ces commandes permettent d'identifier rapidement les appareils présents sur le réseau et de vérifier la configuration IP des postes en mairie.

📸
Invite de commandes Windows — arp -a & ipconfig /all

Quoi capturer : Une fenêtre CMD affichant d'abord arp -a (table ARP avec adresses MAC et IP des équipements réseau) puis ipconfig /all (configuration IP complète du poste).

  1. Win+R → cmd
  2. Tape arp -a et fais une capture
  3. Tape ipconfig /all et fais une capture
  4. Tu peux réaliser cette capture depuis ton propre poste

Scan réseau — Inventaire des appareils

Outil de scan réseau utilisé pour identifier tous les équipements actifs sur le réseau de la mairie avant d'intervenir.

📸
Advanced IP Scanner ou Angry IP Scanner

Quoi capturer : Interface du logiciel de scan réseau après un scan avec la liste des appareils trouvés (adresses IP, noms, constructeurs). Utilise le logiciel sur ton réseau local pour reproduire la capture.

  1. Télécharge Advanced IP Scanner (gratuit)
  2. Lance un scan sur ta plage réseau (ex. 192.168.1.1-254)
  3. Capture la liste des résultats
Semaine 5

RAID, journaux système & dépannage réseau

Activités principales

  • Réparation d'un ancien disque formaté en FAT32.
  • Gestion d'un serveur NAS (accès à distance).
  • Configuration d'un serveur RAID 10 (4 disques, performance + redondance).
  • Analyse des journaux Windows — erreurs Power Kernel et TPM.
  • Intervention sur panne réseau totale dans une commune.
  • Reconfiguration complète des switchs et imprimantes réseau.
Cas concret : Panne réseau totale dans une commune → diagnostic câblage → reconfiguration switch → remise en service des imprimantes réseau et des postes AD en moins d'une journée.

Ce que j'ai appris

  • RAID matériel et logiciel (RAID 1, RAID SHR, RAID 10).
  • Lecture et analyse de journaux système pour diagnostiquer des arrêts brutaux.
  • Dépannage réseau complet en conditions réelles.

Captures à intégrer

Observateur d'événements — Erreurs Power Kernel

Les journaux Windows montrent les erreurs d'arrêt brutal (Power Kernel 41) utilisées pour diagnostiquer les problèmes d'alimentation et de stabilité serveur.

📸
Observateur d'événements Windows — Kernel-Power

Quoi capturer : L'Observateur d'événements filtré sur les erreurs critiques Kernel-Power ID 41 (arrêt brutal inattendu) dans les journaux Système.

  1. Win+R → eventvwr.msc
  2. Journaux Windows → Système
  3. Filtre → ID événement : 41, Source : Kernel-Power
  4. Capture un ou plusieurs événements avec le détail visible

RAID 10 — Gestionnaire de disques

Configuration du serveur RAID 10 : 4 disques en miroir de bandes, offrant à la fois performance et tolérance aux pannes.

📸
Gestionnaire de disques — RAID 10

Quoi capturer : Gestionnaire de disques Windows Server avec les 4 disques du RAID 10 visibles, statut "Sain" et type de volume "RAID-5" ou interface du contrôleur RAID matériel (carte RAID HPE/Dell).

  1. Win+R → diskmgmt.msc
  2. Capture les disques en mode RAID avec leurs statuts
  3. Alternativement : accède à l'interface de gestion du contrôleur RAID matériel (iLO, IDRAC ou BIOS RAID)
Semaine 6

Administration avancée & migration serveur

Activités principales

  • Mise en maintenance d'un serveur (déplacement de baie).
  • Ajout d'imprimantes dans le domaine Active Directory.
  • Supervision de serveurs distants & analyse des erreurs TPM.
  • Intervention mairie (police + école) pour résoudre des problèmes réseau.
  • Grand projet : transfert complet d'un serveur d'entreprise
    • Configuration des machines virtuelles (VM)
    • DNS, passerelles, paramètres réseau
    • Migration Active Directory + données
    • Transfert des rôles FSMO vers le nouveau serveur
    • Travail de nuit pour la bascule (zéro downtime)
    • Création d'un script d'intégration automatique au domaine
Point fort : Migration réalisée de nuit pour ne pas interrompre les utilisateurs — transfert des rôles FSMO validé, Active Directory répliqué sur le nouveau serveur, script PowerShell créé pour rejoindre automatiquement le domaine.

Ce que j'ai appris

  • Organisation d'une migration serveur réelle avec contrainte de continuité de service.
  • Gestion de l'Active Directory à grande échelle (OUs, GPO, utilisateurs).
  • Transfert des rôles FSMO entre contrôleurs de domaine.
  • Automatisation de tâches d'administration avec scripts PowerShell.

Captures à intégrer

Transfert des rôles FSMO — netdom query fsmo

La commande netdom query fsmo confirme que les cinq rôles FSMO ont bien été transférés sur le nouveau contrôleur de domaine après la migration.

📸
CMD Windows Server — netdom query fsmo

Quoi capturer : Résultat de netdom query fsmo sur le nouveau serveur, montrant les 5 rôles FSMO (PDC, RID Master, Infrastructure Master, Schema Master, Domain Naming Master) pointant vers le nouveau DC.

  1. Ouvre CMD en tant qu'administrateur sur le nouveau serveur
  2. Tape : netdom query fsmo
  3. Capture le résultat avec les noms des serveurs détenteurs des rôles

Active Directory — Les deux contrôleurs de domaine

La console ADUC (Active Directory Users and Computers) montrant l'ancien et le nouveau contrôleur de domaine présents dans le domaine pendant la migration.

📸
Console ADUC — Domain Controllers

Quoi capturer : Console "Utilisateurs et ordinateurs Active Directory" → OU "Domain Controllers" avec les deux DC listés. Alternativement, le résultat de nltest /dclist:NOM_DOMAINE.

  1. Ouvre "Utilisateurs et ordinateurs Active Directory" (dsa.msc)
  2. Clique sur "Domain Controllers" dans l'arborescence
  3. Capture les deux contrôleurs de domaine visibles

Script PowerShell — Intégration automatique au domaine

Script créé pour automatiser l'intégration des postes dans le domaine Active Directory, évitant les manipulations manuelles répétitives.

📸
Script PowerShell — Add-Computer

Quoi capturer : Capture du script PowerShell ouvert dans l'ISE ou VS Code, avec le code d'intégration automatique au domaine visible (utilisant Add-Computer, Move-ADObject ou équivalent).

  1. Ouvre PowerShell ISE ou VS Code
  2. Charge le script d'intégration que tu as créé
  3. Capture le code avec la syntaxe visible
  4. Si le script n'est plus disponible, recrée les lignes clés : Add-Computer -DomainName "domaine.local" -Credential (Get-Credential)

Conclusion

Bilan du stage

Ce stage m'a permis d'acquérir une vision complète et réaliste du métier de technicien et d'administrateur systèmes/réseaux en prestataire de services. La variété des interventions — du montage matériel à la migration de serveurs d'entreprise en passant par le dépannage réseau en collectivité — m'a confronté à des situations professionnelles authentiques que les cours seuls ne peuvent simuler.

Point marquant : La migration nocturne du serveur d'entreprise avec transfert des rôles FSMO et création d'un script d'automatisation. Cette mission, réalisée avec une contrainte de zéro interruption de service, m'a démontré l'importance de la planification, de la rigueur et de la gestion du stress en conditions réelles.

Je retiens également la confiance accordée par le tuteur M. Cédric Duval, qui m'a rapidement intégré aux interventions clients et m'a laissé une autonomie croissante au fil des semaines. Ce stage a confirmé et renforcé mon intérêt pour l'administration systèmes et réseaux.
