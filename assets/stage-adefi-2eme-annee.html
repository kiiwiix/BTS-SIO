Semaine 1

Infrastructure serveur & virtualisation

Activités principales

  • Analyse d'un problème de scan réseau sur imprimante OKI (erreur de stockage).
  • Vérification du chemin de stockage des scans sur le serveur et des droits utilisateurs.
  • Tests avec activation / désactivation du pare-feu Windows Server.
  • Mise en place de deux volumes RAID 1 (miroir) sur un serveur HPE.
  • Installation d'une clé HPE pour l'hyperviseur.
  • Installation et configuration de VMware ESXi sur le serveur physique.
Diagnostic imprimante : L'erreur "Erreur de stockage" sur l'OKI était causée par une incompatibilité de sécurité entre l'imprimante et le serveur (partage SMB trop restrictif). Solution : ajustement des droits sur le dossier de destination des scans.
Résultat ESXi : Hyperviseur VMware ESXi opérationnel sur le serveur HPE, prêt à héberger les VMs des clients — avec les deux volumes RAID 1 configurés pour la redondance des données.

Captures à intégrer

VMware ESXi — Interface de gestion

L'interface web de VMware ESXi montre le serveur HPE virtualisé, prêt à héberger des machines virtuelles pour les clients d'Adefi.

📸
Interface ESXi Host Client

Quoi capturer : Interface web ESXi (https://IP_SERVEUR/ui) avec le tableau de bord principal : CPU, RAM, stockage disponibles, et les VMs créées listées.

  1. Accède à l'interface ESXi Host Client via le navigateur
  2. Capture le tableau de bord principal (Summary) avec les ressources du serveur
  3. Capture également la liste des datastores (avec les deux volumes RAID 1)

Diagnostic imprimante OKI — Configuration réseau

Capture de la configuration réseau de l'imprimante OKI ou de l'interface de paramétrage du scan vers dossier réseau, montrant la résolution du problème.

📸
Interface web OKI — Paramètres de scan réseau

Quoi capturer : Interface web de l'imprimante OKI → Section "Scan vers dossier réseau" ou "SMB" avec le chemin de destination configuré. Alternativement, capture les droits NTFS du dossier de stockage sur le serveur (onglet Sécurité des propriétés du dossier).

  1. Accède à l'interface web de l'imprimante OKI (IP de l'imprimante)
  2. Navigue vers les paramètres de scan réseau
  3. Ou : clic droit sur le dossier de stockage → Propriétés → Onglet Sécurité
  4. Capture les droits attribués
Semaine 2

Tests matériels & messagerie

Activités principales

  • Présentation et tests d'un onduleur professionnel Nitram.
  • Tests dans l'infrastructure serveur : alimentation, supervision locale, remontée d'informations SNMP.
  • Configuration de la messagerie POP / IMAP sur serveur et postes utilisateurs.
  • Diagnostic d'un ralentissement d'Outlook au démarrage sur certains postes.
Analyse coût/performance : La carte réseau SNMP de l'onduleur Nitram offre une supervision complète mais à un coût prohibitif pour une petite structure. Conclusion : solution non adaptée au budget, alternative logicielle recommandée.

Ce que j'ai appris

  • Fonctionnement d'un onduleur professionnel et supervision via SNMP.
  • Paramétrage des comptes POP / IMAP et configuration Outlook.
  • Analyse coût / performance d'une solution IT professionnelle.

Captures à intégrer

Onduleur Nitram — Interface de supervision

Interface de supervision locale de l'onduleur montrant les paramètres d'alimentation, l'autonomie batterie et l'état du réseau électrique.

📸
Interface de supervision onduleur Nitram

Quoi capturer : Interface web ou logiciel de supervision de l'onduleur Nitram affichant : tension entrée/sortie, charge %, autonomie batterie estimée, historique des coupures de courant.

  1. Accède à l'interface de supervision de l'onduleur (IP locale ou logiciel dédié)
  2. Capture le tableau de bord principal avec les métriques électriques
  3. Si plus d'accès : prends une photo de l'affichage LCD frontal de l'onduleur

Configuration Outlook — Compte POP/IMAP

Paramétrage d'un compte de messagerie professionnel dans Outlook avec les paramètres POP ou IMAP du serveur mail de l'entreprise cliente.

📸
Outlook — Paramètres du compte de messagerie

Quoi capturer : Fenêtre des paramètres du compte Outlook : Fichier → Paramètres du compte → onglet "Serveur entrant / sortant" avec les valeurs IMAP/POP3 et SMTP configurées (masque les mots de passe).

  1. Ouvre Outlook → Fichier → Paramètres du compte
  2. Double-clique sur le compte à capturer
  3. Paramètres supplémentaires → Serveur sortant / Options avancées
  4. Masque le mot de passe et capture l'écran
Semaine 3

VPN & préparation de postes

Activités principales

  • Création d'un compte utilisateur VPN sur routeur Zyxel.
  • Configuration des accès pour TSE (Terminal Server / Services).
  • Test complet de la connexion VPN + accès Bureau à distance.
  • Montage complet d'un ordinateur (assemblage matériel).
  • Installation de Windows 11 Pro avec activation de licence.
  • Transfert de fichiers entre machines via Linux Mint (contournement des restrictions Windows).
Technique de contournement : Les restrictions NTFS de Windows bloquaient le transfert de fichiers entre les deux machines. Solution : boot sur Linux Mint depuis une clé USB pour accéder aux fichiers sans les contraintes de permissions Windows.

Ce que j'ai appris

  • Configuration d'un VPN utilisateur sur routeur Zyxel (L2TP/IPSec ou SSL-VPN).
  • Installation et déploiement propre de Windows 11 Pro avec licence OEM.
  • Utilisation de Linux comme outil de dépannage pour les problèmes de permissions Windows.

Captures à intégrer

VPN Zyxel — Compte utilisateur créé

Interface d'administration du routeur Zyxel montrant le compte VPN créé et la connexion testée avec succès depuis un poste extérieur.

📸
Interface Zyxel — Gestion VPN / Comptes utilisateurs

Quoi capturer : Interface web du routeur Zyxel → Section VPN → Comptes utilisateurs VPN. On doit voir le compte créé avec le nom d'utilisateur et le type d'accès (SSL-VPN ou L2TP).

  1. Accède à l'interface d'administration du routeur Zyxel
  2. VPN → SSL VPN (ou L2TP) → User Accounts
  3. Capture la liste des utilisateurs VPN avec le nouveau compte visible

Connexion Bureau à distance (RDP) — Test réussi

La connexion Bureau à distance fonctionne via le tunnel VPN, permettant d'accéder au serveur TSE depuis l'extérieur du réseau d'entreprise.

  1. Établis la connexion VPN depuis un poste externe
  2. Lance mstsc.exe et connecte-toi au serveur TSE
  3. Capture la session RDP ouverte (masque les données sensibles si besoin)

Confirmation de l'activation de Windows 11 Pro après installation propre, avec la licence valide associée au matériel.

  2. Capture l'écran montrant l'activation confirmée
  3. Capture également Système → À propos pour voir le nom du PC et la version
Semaine 4

Administration Linux & audit sécurité réseau

Activités principales

  • Administration d'une machine Linux :
    • Création d'un utilisateur principal adefi.
    • Suppression du compte temporaire OEM.
    • Configuration d'un environnement Linux sécurisé (droits sudo).
  • Audit réseau avec l'outil GEPRECIA.
  • Connexion à un Raspberry Pi en SSH pour administration distante.
  • Analyse des requêtes DNS du réseau (flux entrants/sortants).
  • Observation du comportement de l'antivirus ESET sur le réseau.
Problème rencontré : L'utilisateur adefi nouvellement créé n'avait pas les droits administrateur. Solution : Ajout au groupe sudo (usermod -aG sudo adefi), vérification des privilèges, puis suppression sécurisée du compte OEM.

Ce que j'ai appris

  • Gestion des utilisateurs et des droits Linux (commandes useradd, usermod, userdel).
  • Administration SSH d'équipements distants (Raspberry Pi).
  • Analyse de sécurité réseau : requêtes DNS et comportement antivirus.

Captures à intégrer

Terminal Linux — Gestion utilisateurs & sudo

Séquence de commandes Linux : création de l'utilisateur adefi, ajout au groupe sudo, vérification des droits, suppression du compte OEM.

📸
Terminal Linux — useradd / usermod / userdel

Quoi capturer : Terminal Linux avec les commandes sudo useradd -m adefi, sudo usermod -aG sudo adefi, id adefi (pour voir les groupes) et sudo userdel -r oem.

  1. Ouvre un terminal sur ta VM Linux (Ubuntu/Debian)
  2. Tape les commandes une par une et capture le résultat
  3. sudo useradd -m adefi && sudo passwd adefi
  4. sudo usermod -aG sudo adefi
  5. id adefi — vérifie la présence dans sudo
  6. Capture l'ensemble des commandes

Connexion SSH — Raspberry Pi

Connexion SSH depuis un poste vers le Raspberry Pi utilisé comme équipement de supervision réseau ou serveur léger.

📸
Terminal — ssh pi@IP_RASPBERRY

Quoi capturer : Terminal (Linux ou Windows Terminal / PuTTY) avec une session SSH ouverte sur le Raspberry Pi. Le prompt du Raspberry doit être visible (pi@raspberrypi:~$ ou similaire).

  1. Depuis un terminal : ssh pi@192.168.X.X (adresse IP du Raspberry Pi)
  2. Accepte la clé d'hôte si c'est la première connexion
  3. Une fois connecté, tape uname -a ou neofetch pour afficher les infos système
  4. Capture le terminal avec la session SSH active

GEPRECIA — Audit réseau

Interface ou rapport de l'outil d'audit GEPRECIA montrant les équipements identifiés sur le réseau du client audité.

📸
Interface GEPRECIA ou rapport d'audit réseau

Quoi capturer : Interface de GEPRECIA avec les résultats de l'audit : équipements détectés, services ouverts, vulnérabilités identifiées. Si GEPRECIA n'est plus disponible, capture un scan Nmap équivalent : nmap -sV 192.168.X.0/24

  1. Lance GEPRECIA ou un outil équivalent (Nmap, Masscan)
  2. Effectue un scan sur la plage réseau cible
  3. Capture les résultats avec les équipements listés
Semaine 5

Déploiement Active Directory

Activités principales

  • Passage du réseau en profil entreprise sur le nouveau serveur.
  • Scan réseau pour identifier l'adresse iLO du serveur HPE.
  • Désactivation du Secure Boot pour permettre l'installation.
  • Ajout du serveur au domaine Active Directory existant.
  • Promotion du serveur en contrôleur de domaine (DC).
  • Installation des rôles DNS et DHCP.
  • Vérification de la réplication Active Directory entre les DC.
  • Contrôle des GPO et scripts de démarrage.
Résultat : Nouveau contrôleur de domaine opérationnel, réplication AD/DNS vérifiée entre les deux serveurs, services DNS et DHCP configurés et distribués sur le réseau.

Captures à intégrer

Active Directory — Contrôleur de domaine promu

La console ADUC (Active Directory Users and Computers) montre le nouveau serveur promu en contrôleur de domaine, visible dans l'OU "Domain Controllers".

📸
Console ADUC — Domain Controllers

Quoi capturer : Console "Utilisateurs et ordinateurs Active Directory" (dsa.msc) avec l'OU "Domain Controllers" ouverte, montrant les contrôleurs de domaine présents dans le domaine.

  1. Sur le serveur DC : Win+R → dsa.msc
  2. Clique sur "Domain Controllers" dans l'arborescence gauche
  3. Capture les serveurs listés (les deux DCs si migration)

Console DNS — Zones configurées

La console DNS montre les zones de recherche directe et inversée configurées pour le domaine Active Directory, avec les enregistrements des contrôleurs de domaine.

📸
Gestionnaire DNS — Zones du domaine

Quoi capturer : Console de gestion DNS (dnsmgmt.msc) avec la zone du domaine ouverte, montrant les enregistrements A des contrôleurs de domaine et les enregistrements SRV AD.

  1. Win+R → dnsmgmt.msc
  2. Développe le serveur DNS → Zones de recherche directe
  3. Clique sur la zone du domaine (ex. adefi.local)
  4. Capture les enregistrements DNS visibles

Réplication AD — dcdiag ou repadmin

La commande dcdiag ou repadmin /replsummary vérifie que la réplication entre les deux contrôleurs de domaine fonctionne correctement sans erreurs.

📸
CMD Windows Server — repadmin /replsummary

Quoi capturer : Résultat de repadmin /replsummary ou dcdiag /test:replications montrant "0 échec(s)" pour la réplication entre les contrôleurs de domaine.

  1. Ouvre CMD en tant qu'administrateur sur un DC
  2. Tape : repadmin /replsummary
  3. Ou : dcdiag /test:replications /v
  4. Capture le résultat avec les statuts de réplication
Semaine 6

Migration de données & sauvegarde Veeam

Activités principales

Migration de données :

  • Transfert des rôles FSMO vers le nouveau serveur.
  • Migration d'environ 182 Go de données d'entreprise.
  • Montage et formatage d'un disque 1 To.
  • Hébergement des données sur le nouveau serveur Data.

Services & virtualisation :

  • Mise en place du TSE (Terminal Services).
  • Configuration RDP / Bureau à distance.
  • Mise en place de Veeam Backup (sauvegarde complète + réplication VMs).
  • Virtualisation d'un poste avec OpenBee.
Défi : Assurer la continuité de service pendant la migration des 182 Go de données. Solution : migration progressive par lots, vérification de l'intégrité à chaque étape, bascule planifiée hors heures de bureau.

Ce que j'ai appris

  • Migration de données serveur en maintenant la continuité de service.
  • Mise en place de sauvegardes professionnelles avec Veeam Backup & Replication.
  • Utilisation de la virtualisation en environnement de production (ESXi + OpenBee).

Captures à intégrer

Veeam Backup — Jobs de sauvegarde configurés

L'interface Veeam Backup & Replication montre les jobs de sauvegarde créés pour les VMs et les serveurs physiques, avec les planifications et les statuts.

📸
Console Veeam Backup — Liste des jobs

Quoi capturer : Interface Veeam Backup & Replication avec la liste des jobs de sauvegarde configurés : nom du job, machines concernées, planification (quotidienne/hebdomadaire) et dernier statut (Success/Warning).

  1. Ouvre la console Veeam Backup & Replication
  2. Onglet "Jobs" ou "Backup" dans le panneau gauche
  3. Capture la liste complète des jobs avec leur statut
  4. Double-clique sur un job pour capturer les détails (machines, rétention, planification)

Transfert des rôles FSMO — Nouveau serveur

La commande netdom query fsmo confirme que les cinq rôles FSMO ont été transférés vers le nouveau serveur Data lors de la migration.

📸
CMD — netdom query fsmo

Quoi capturer : Résultat de netdom query fsmo sur le nouveau serveur, montrant les 5 rôles FSMO (PDC, RID, Infrastructure, Schema, Domain Naming) avec le nom du nouveau serveur détenteur.

  1. Ouvre CMD en tant qu'administrateur sur le nouveau serveur
  2. Tape : netdom query fsmo
  3. Capture le résultat complet

Nouveau serveur Data — Données migrées

L'Explorateur Windows sur le nouveau serveur montre le disque 1 To formaté avec les données migrées (182 Go) accessibles et partagées sur le réseau.

📸
Explorateur Windows — Disque de données + Partages réseau

Quoi capturer : Explorateur Windows sur le serveur Data montrant : (1) le disque 1 To avec l'espace occupé visible, (2) la gestion des partages (fsmgmt.msc) avec les dossiers partagés sur le réseau.

  1. Ouvre l'Explorateur Windows sur le nouveau serveur Data
  2. Capture le disque 1 To avec la taille utilisée (~182 Go)
  3. Win+R → fsmgmt.msc pour les partages réseau
  4. Capture les dossiers partagés accessibles aux utilisateurs

Conclusion

Bilan du stage de 2ème année

Ce deuxième stage chez Adefi Informatique m'a permis d'aborder des problématiques techniques nettement plus complexes qu'en 1ère année. La virtualisation avec VMware ESXi, le déploiement complet d'un Active Directory avec ses rôles DNS/DHCP, la mise en place de Veeam Backup et la migration de données en production m'ont confronté aux enjeux réels de l'administration systèmes en entreprise.

Points forts : Le déploiement du nouveau contrôleur de domaine avec transfert des rôles FSMO, et la mise en place de Veeam Backup qui protège désormais l'ensemble des VMs du client — deux missions à fort impact opérationnel réalisées en autonomie croissante.

Ce stage confirme ma montée en compétences progressive sur les deux années de BTS SIO et consolide mon projet professionnel orienté administration systèmes, réseaux et virtualisation. La confiance accordée par M. Cédric Duval, qui m'a laissé piloter des migrations de production, m'a apporté une expérience et une assurance techniques considérables.
